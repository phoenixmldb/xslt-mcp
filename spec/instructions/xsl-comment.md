# xsl:comment

Creates an XML comment node in the result tree.

## Attributes

| Attribute | Required | Type | Description |
|-----------|----------|------|-------------|
| `select` | No | `expression` | Expression producing the comment content. Added in XSLT 3.0. |

## Content Model

Sequence constructor (used to produce the comment content if `select` is absent).

## Description

`xsl:comment` creates a comment node in the result document. The content of the comment is determined by the `select` attribute or by evaluating the sequence constructor body. The resulting string becomes the text between `<!--` and `-->` in the serialized output.

If the content contains the string `--` (double hyphen) or ends with `-`, the processor must either signal an error or insert a space to make the comment well-formed, depending on the implementation.

The `select` attribute was added in XSLT 3.0 as a convenience. When both `select` and a body are present, it is a static error.

## Examples

### Static Comment

```xml
<xsl:comment> Generated by XSLT on <xsl:value-of select="current-dateTime()"/> </xsl:comment>
```

### Using Select

```xml
<xsl:comment select="'Generated from: ' || document-uri(/)"/>
```

## Error Codes

- **XTTE3170**: The content contains `--` or ends with `-`.
- **XTSE0010**: Both `select` and a sequence constructor body are present.
