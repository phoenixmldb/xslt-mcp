# XPDY0002

**Type:** Dynamic error (XPath)

The context item is absent (undefined) when it is required. Raised when an expression references `.`, `position()`, `last()`, or other context-dependent constructs but no context item has been set.

## Common Causes

- Using `.` in a global variable or function where no context item exists.
- Calling `fn:string()`, `fn:number()`, or `fn:normalize-space()` with no argument when no context item is set.
- Using `position()` or `last()` outside of a sequence processing context.

## Example

```xml
<!-- Global variable with no source document -->
<xsl:variable name="root" select="."/>  <!-- XPDY0002 if no context item -->

<xsl:function name="my:func" as="xs:string">
  <xsl:sequence select="string()"/>  <!-- XPDY0002: no context item in functions -->
</xsl:function>
```
